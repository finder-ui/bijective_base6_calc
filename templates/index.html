A Primer on Numerical Systems

Most number systems we use are positional. This means the value of a digit depends on its position within a number. The "base" of a system tells us how many unique digits are used. The value of each position is an exponent of the base, starting from 0 on the right.

However, the inclusion of zero introduces certain ambiguities, or representational redundancy. For instance, the integer 'six' can be represented as 6, 06, or 006. While this is a trivial issue in human arithmetic, it can complicate computational processes that require a single, canonical representation for every number.

Understanding the Bijective System

This application explores the senary bijective system, a base-6 numeration that uses the digit set D = {1, 2, 3, 4, 5, 6}. Its key feature is establishing a bijective (one-to-one) correspondence between the set of positive integers (1, 2, 3, ...) and the set of all possible finite strings of its digits.

Core Properties

  • No Zero: The most obvious difference is the lack of a zero digit. This eliminates the need for a placeholder and removes the ambiguity of leading zeros.
  • Lexicographical Order: The numerical order of the integers corresponds perfectly to the lexicographical (dictionary) order of their string representations. For example, 12 is numerically smaller than 13, and the string "12" comes before "13" in the dictionary. This is not always true in systems with zero (e.g., `6` is less than `11`, but "6" comes after "11" in dictionary order).

The Conversion Algorithm Explained

The trick to converting from decimal to bijective base-6 is to temporarily shift the number system to align with standard computer arithmetic. Computers naturally work with remainders from 0 to 5, but our digits are 1 to 6. We bridge this gap by subtracting 1 before each calculation.

Let's convert the decimal number 43:

  1. Step 1: Start with 43. Subtract 1: 43 - 1 = 42. Now, find the remainder when dividing by 6. 42 % 6 = 0. The 0th digit in our character set "123456" is '1'. Our first result digit is 1. The new number for the next step is floor(42 / 6) = 7.
  2. Step 2: Start with 7. Subtract 1: 7 - 1 = 6. Now, find the remainder. 6 % 6 = 0. The 0th digit is again '1'. Our second result digit is 1. The new number is floor(6 / 6) = 1.
  3. Step 3: Start with 1. Subtract 1: 1 - 1 = 0. Now, find the remainder. 0 % 6 = 0. The 0th digit is again '1'. Our third result digit is 1. The new number is floor(0 / 6) = 0.
  4. Step 4: The number is 0, so we stop. Reverse the digits we found: 111.

Thus, the decimal number 43 is represented as 111 in bijective base-6.

Practical Applications

Bijective numeration is more than a curiosity; it solves real-world problems in computer science.

  • Combinatorics & Enumeration: The system provides a natural mapping for problems involving sequences. For example, it can uniquely number every possible sequence of rolls of a six-sided die. A sequence of three rolls (e.g., 4, 2, 5) corresponds directly to the bijective number 425, providing a compact and unique integer ID for every outcome.
  • Spreadsheet Columns: Microsoft Excel and Google Sheets use a bijective base-26 system (A-Z) for columns. After Z comes AA, then AB, AC, and so on. This is a perfect, intuitive real-world example where there is no "zeroth" column.
  • URL Shorteners: Services like bit.ly need to create the shortest possible unique string for billions of links. A bijective base-62 system (a-z, A-Z, 0-9) can represent a massive database ID like 2,183,401,055,849 as the short, clean string `short`.

Advanced Concepts

The properties of bijective numeration extend into more complex areas of computer science and mathematics:

  • Canonical Forms in Cryptography: In advanced cryptography like Elliptic Curve Cryptography (ECC), it is absolutely critical that every unique public key has one, and only one, valid byte representation. If two different strings could represent the same key, it could allow an attacker to create a different-looking but valid signature for a transaction, leading to bugs or security vulnerabilities. Bijective systems provide a mathematical guarantee of this canonical representation.
